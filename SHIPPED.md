### What Changed

**Before v1.4.0:**

- Users had to manually run `/speckit.plan` for each feature (1-2 hours)

## How It Works

### Configuration Flow

**Step 1**: User runs `/stackshift.start`

**Step 2**: Gear 1 asks questions:
- Question 1: Route? (Greenfield/Brownfield)
- Question 2: Upgrade? (if Brownfield)
- Question 3: Transmission? (Manual/Cruise Control)
- **Question 4: Thoroughness?** üÜï (Specs | Specs+Plans | Specs+Plans+Tasks)
- Question 5: Clarifications? (if Cruise Control)
- Question 6: Implementation scope? (if Cruise Control)

**Step 3**: Configuration saved to `.stackshift-state.json`:
```json
{
  "config": {
    "gear3_thoroughness": "specs+plans+tasks"
  }
}
```

**Step 4**: Gear 3 executes based on config:
- **Always**: Generate all specs
- **If Level 2+**: Auto-generate plans for PARTIAL/MISSING (parallel, 5 at a time)
- **If Level 3**: Auto-generate comprehensive tasks (parallel, 3 at a time)
