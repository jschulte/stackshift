## How It Works

### Step 1: StackShift Gear 2 (Reverse Engineering)

When developers run `/stackshift:reverse-engineer`, the **stackshift-code-analyzer** agent now generates `test-specifications.md` with:

**For each business rule/conditional:**
- Unique test ID (TEST-001, TEST-002, etc.)
- Clear description
- Given-When-Then format
- Exact test data
- Expected assertions
- Source reference (which reverse-engineering doc)
- Edge cases

**For integrations:**
- Success scenarios with mock setup
- Error scenarios (404, 500, timeout)
- API call parameters
- Expected responses

**For components:**
- Rendering tests with props
- Conditional rendering logic
- User interaction tests
- Accessibility requirements

### Step 2: Developer Uses AI to Generate Code

**Developer workflow:**
1. Open `docs/reverse-engineering/test-specifications.md`
2. Copy a test spec (e.g., TEST-001)
3. Paste into Claude/Copilot with prompt:
   ```
   Generate Jest test code from this specification:
   [paste spec]
   ```
4. Review generated code
5. Run tests

**AI generates working code like:**
```javascript
describe('Disclaimer Display Logic', () => {
  it('should display full text for disclaimers ≤200 characters', () => {
    // Arrange
    const disclaimer = {
      type: 'state',
      text: 'This disclaimer is under 200 characters.'
    };

    // Act
    const result = shouldShowFullText(disclaimer);

    // Assert
    expect(result).toBe(true);
  });
});
```
